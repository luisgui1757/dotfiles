> Published 2026-07-21 as immutable GitHub release `357094679`. Annotated tag
> object `1539e550ac45d0a9732f329cb1ae3fb13bb078a8` peels to commit

## Highlights

- Herdr's expanded agent cards restore the previous two-line layout: the status
  icon plus workspace/tab on the first line, explicit `idle` or `working` plus
  the agent name on the second, and the separating blank line between cards.
  The POSIX and Windows managed configs retain byte-identical layout semantics.
- Git now enables `fetch.prune = true` through the managed XDG global config,
  so successful fetches remove stale remote-tracking branches by default. The
  user-owned `~/.gitconfig` remains untouched and can override this default.
- The reviewed Homebrew installer commit and Renovate validator runtime pin are
  refreshed to the merged dependency identities.

## Compatibility and upgrade

This is a backward-compatible minor release: it restores user-visible Herdr
state text and card spacing and adds automatic fetch pruning without changing
the public setup commands or managed config shapes. No breaking contract change
is introduced.

Do not update an older checkout in place. Clone the exact annotated `v0.4.0`
tag beside the checkout that currently owns live config, then run:

```bash
./setup.sh --all
```

```powershell
.\setup.ps1 -All
```

The same entrypoints support a direct side-by-side v0.1.0 migration through the
versioned frozen-source transaction. Existing v0.2.0 and v0.3.0 installations
use ordinary setup/update reconciliation from the new checkout. Keep the prior
checkout and any printed recovery directory until setup completes and the
applications you use have been checked. An unfinished v0.2.0 or v0.3.0
recovery must be accepted or rolled back from its retained exact release
checkout first; v0.4.0 setup refuses to bypass either transaction.

## Release identity

The POSIX prerequisite helper and both v0.1.0 migrators require the tag name
`v0.4.0` and accept only the exact local annotated tag object, peeled commit,
clean HEAD, and matching official-remote identities. Only the explicit
`--allow-unreleased` POSIX field-test lane may instead accept a clean current
official branch head; that lane is not migration authority.
