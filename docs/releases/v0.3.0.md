# v0.3.0 — Pi Rose Pine themes and resilient cross-platform setup

> Release candidate notes. Publish these only with the official annotated
> `v0.3.0` tag after the deterministic evidence gate below passes.

## Highlights

- Pi now installs three canonical Fable-tuned Rose Pine themes: Main
  (`rose-pine`, still the first-setup default), Moon, and Dawn. Setup preserves
  a selected managed variant, safely retires recognized trial aliases, and
  keeps the exact multiline-input keybindings for Shift+Enter and Ctrl+J.
- Herdr uses its corrected indexed-workspace navigation across supported
  platforms. Native Windows opens managed panes with PowerShell 7 so the repo's
  profile, history prediction, completion, and prompt load consistently.
- POSIX Nix bootstrap now avoids upstream shell-profile mutation, skips the
  unused mutable channel, repairs restrictive store modes left by interrupted
  daemon installs, and retains the explicit verified branch-head field-test
  lane without weakening the exact-release default.
- Pinned tooling moves to Pi `0.80.10`, Tree-sitter CLI `0.26.11`, gh-dash
  `v4.25.2`, lazygit `v0.63.1`, Windows Terminal portable `v1.24.11911.0`,
  `setuptools` `83.0.0`, Renovate `43.270.0`, and the reviewed installer
  commits/hashes recorded in the supply-chain ledger.

## Compatibility and upgrade

This is a backward-compatible minor release: it adds user-visible Pi theme
functionality and Herdr behavior while retaining the public setup commands and
managed config shapes. No breaking contract change is introduced.

Do not update an older checkout in place. Clone the exact annotated `v0.3.0`
tag beside the checkout that currently owns live config, then run:

```bash
./setup.sh --all
```

```powershell
.\setup.ps1 -All
```

The same entrypoints support a direct side-by-side v0.1.0 migration through the
versioned frozen-source transaction. Existing v0.2.0 installations use the
ordinary setup/update reconciliation from the new checkout. Keep the prior
checkout and any printed recovery directory until the new setup completes and
the applications you use have been checked.
An unfinished v0.2.0 recovery must be accepted or rolled back from its retained
exact v0.2.0 checkout first; v0.3.0 setup refuses to bypass it.
