# v0.4.1 — Conflict-free Herdr workspace jumps

> Published 2026-07-22 as immutable GitHub release `357785004`. Annotated tag
> object `558d19a8c62453f68e5463e8999b216e0b692551` peels to commit
> `bac8cc97177b3bb58119fde5720b31e6b57febcc`.

## Highlights

- Direct Herdr workspace jumps move from `Prefix`, then `Shift+1..9` to
  `Prefix`, then `Ctrl+Alt+1..9`. On macOS that is `Ctrl+B`, release, then
  `Control+Option+1..9`. This preserves indexed workspace navigation without
  colliding with `Prefix+$`, which remains workspace rename.
- `Prefix+g` and `Prefix+w` remain the searchable full workspace/tab/pane
  navigator, while `Prefix+Up/Down` remains sequential workspace navigation.
- The Linux/Home Manager setup regression now uses syntax accepted by macOS's
  system Bash 3.2 and checks the guarded-profile fixture exit status, preventing
  that release-gate path from emitting a parse error and returning a false pass.

## Compatibility and upgrade

This is a backward-compatible patch release. It fixes one conflicting Herdr
shortcut and one test-fixture portability defect; setup commands, managed data
shapes, and application ownership remain unchanged.

Do not update an older checkout in place. Clone the exact annotated `v0.4.1`
tag beside the checkout that currently owns live config, then run:

```bash
./setup.sh --all
```

```powershell
.\setup.ps1 -All
```

The same entrypoints support a direct side-by-side v0.1.0 migration through the
versioned frozen-source transaction. Existing v0.2.0, v0.3.0, and v0.4.0
installations use ordinary setup/update reconciliation from the new checkout.
Keep the prior checkout and any printed recovery directory until setup completes
and the applications you use have been checked. An unfinished v0.2.0, v0.3.0,
or v0.4.0 recovery must be accepted or rolled back from its retained exact
release checkout first; v0.4.1 setup refuses to bypass any earlier transaction.
